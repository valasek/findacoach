## Project Overview

**Find a Coach** is a solo-developer Ruby on Rails 8 SaaS application that helps coaches manage clients, coaching sessions, notes, and services. Deployed as a Docker container via Kamal.

### Tech Stack

| Layer | Technology |
|---|---|
| Framework | Ruby on Rails 8 |
| Database | SQLite (via `solid_cache`, `solid_queue`, `solid_cable`) |
| Frontend | Hotwire (Turbo + Stimulus), Tailwind CSS, DaisyUI |
| Auth | Devise + OmniAuth (Google OAuth2) |
| Admin | Avo (restricted to owner email) |
| Pagination | Kaminari |
| Jobs | Solid Queue |
| Mailer | Action Mailer → Resend |
| Error tracking | Sentry |
| Deployment | Kamal (Docker) |
| Style linting | RuboCop Rails Omakase |
| Security scan | Brakeman |
| Testing | Minitest + Capybara |

### Domain Models

```
User           — Devise-managed coach account (email, provider/uid for OAuth)
  └── UserProfile  — username, bio, full_name, phone, website
  └── Client[]     — coach's clients (name, email, phone, notes)
       └── Session[] — coaching sessions (date, duration, notes, paid, group, group_size)
  └── Service[]    — session types offered by coach (name, default flag)

ApplicationData  — singleton table (login_to_demo_count)
Changelog        — loaded from config/data/changelog.yml (not ActiveRecord)
```

### Key Schema Details

- `sessions`: date, duration (decimal 3,1), notes (text), paid (bool, default true), group (bool), group_size (int), belongs_to client + service
- `services`: name (string, unique per user), default (bool, unique true per user via partial index)
- `user_profiles`: username (unique), bio, full_name, phone, website

### Routes Pattern

```ruby
resources :clients do
  resources :sessions       # /clients/:client_id/sessions
end
resources :users do
  resources :services       # /users/:user_id/services
end
resources :sessions, only: [:new, :create]   # standalone session start
get "session/start", to: "sessions#new"
get ":username", to: "findacoach#coach_homepage"  # public profile — catch-all, must be last
```

### Project Structure

```
app/controllers/          # RESTful controllers (thin)
app/models/               # ActiveRecord models + concerns
app/views/                # ERB templates; shared partials in views/partials/
app/javascript/controllers/  # Stimulus controllers
app/mailers/              # Action Mailer (sends via Resend)
app/services/             # Plain Ruby service objects (only when truly needed)
config/routes.rb          # All routes
config/data/changelog.yml # Changelog content (not in DB)
db/schema.rb              # READ ONLY — auto-generated by Rails
db/migrate/               # Create new; never edit existing
```

### Deployment & Environment

- **Dev**: `bin/dev` (Foreman: Puma + Tailwind watcher via `Procfile.dev`)
- **Production**: Docker container deployed with `kamal deploy`, SQLite, Thruster
- **Email**: Action Mailer → Resend (`RESEND_API_KEY` in credentials)
- **Errors**: Sentry (`sentry-ruby`, `sentry-rails`)
- **Assets**: Propshaft (no Webpacker/Sprockets)
- **Credentials**: `VISUAL="code --wait" bin/rails credentials:edit`
